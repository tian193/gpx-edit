# GPX编辑器三项功能改善设计文档

> 创建日期: 2026-05-18
> 状态: 设计确认中

## 一、功能概述

本次改善包含三个功能模块：
1. 数据列表列配置系统 — 支持自定义显示列和计算列
2. 移动航点对话框 — 三种移动模式，内置CGCS2000坐标转换
3. 地图交互编辑 — MapSource风格的地图编辑体验 + 天地图集成

---

## 二、功能一：数据列表列配置

### 2.1 需求描述

当前树形列表固定显示4列（类型、名称、纬度、经度），需要改为可配置的列系统：
- 支持选择显示/隐藏哪些属性列
- 支持列顺序拖动调整
- 支持"计算列"（如CGCS2000投影坐标）
- 配置全局保存，所有文件共用

### 2.2 列类型定义

**原生列（GPX文件自带属性）：**
| 列ID | 显示名 | 数据来源 |
|------|--------|----------|
| type | 类型 | wpt.type |
| name | 名称 | wpt.name |
| lat | 纬度 | wpt.latitude |
| lon | 经度 | wpt.longitude |
| ele | 海拔 | wpt.elevation |
| time | 时间 | wpt.time |
| desc | 描述 | wpt.description |
| cmt | 备注 | wpt.comment |
| sym | 符号 | wpt.symbol |
| source | 来源 | wpt.source |

**计算列（实时计算）：**
| 列ID | 显示名 | 计算方式 |
|------|--------|----------|
| cgcs2000_x | CGCS2000 X | WGS84→CGCS2000投影转换（东向坐标） |
| cgcs2000_y | CGCS2000 Y | WGS84→CGCS2000投影转换（北向坐标） |

### 2.3 CGCS2000投影坐标计算

- 投影方式：高斯-克吕格投影（6度带）
- 带号确定：根据第一个航点的经度自动计算
  - 带号 = floor(经度 / 6) + 1
  - 中央经线 = 带号 × 6 - 3
- 坐标系：CGCS2000（与WGS84椭球参数几乎相同，差异<0.1mm）
- 转换库：使用pyproj库实现WGS84经纬度→CGCS2000投影坐标
- 列头显示：带号信息，如"CGCS2000 X (39带)"

### 2.4 列配置对话框

**布局：**
- 左侧面板：可用列列表（未显示的列）
  - 原生列和计算列分组显示
  - 点击添加到右侧已显示列表
- 右侧面板：已显示列列表
  - 支持拖动调整顺序
  - 点击移除回到左侧
- 底部按钮：确定、取消、重置默认

**交互：**
- 点击列头区域的"⚙ 列配置"按钮打开对话框
- 对话框打开时预览效果实时更新
- 确定后保存配置并刷新数据列表

### 2.5 配置持久化

配置保存到 `~/.gpx_editor/column_config.json`：

```json
{
  "visible_columns": ["type", "name", "lat", "lon", "ele", "time"],
  "column_order": ["type", "name", "lat", "lon", "ele", "time", "desc", "cmt", "sym", "source", "cgcs2000_x", "cgcs2000_y"],
  "column_widths": {
    "type": 60,
    "name": 120,
    "lat": 100,
    "lon": 100
  }
}
```

### 2.6 实现要点

- 新建 `gpx_editor/gui/column_config_dialog.py` — 列配置对话框
- 新建 `gpx_editor/core/coord_converter.py` — 坐标转换模块（CGCS2000）
- 修改 `gpx_editor/gui/main_window.py` — 集成列配置系统
- 依赖：pyproj库（requirements.txt中添加）

## 三、功能二：移动航点（三种模式）

### 3.1 需求描述

重做移动航点对话框，支持三种移动模式：
1. 手动输入坐标（支持经纬度和CGCS2000投影坐标）
2. 地图选点移动
3. 精细移动（X/Y偏移量，单位米）

### 3.2 对话框设计

**整体布局：**
- 标题栏：移动航点 — [航点名称]
- 当前位置显示区：同时显示WGS84经纬度和CGCS2000投影坐标
- 标签页切换：手动输入 / 地图选点 / 精细移动
- 底部按钮：确定、取消

### 3.3 标签页1：手动输入

**功能：**
- 支持两种输入模式切换：经纬度（WGS84）/ 投影坐标（CGCS2000）
- 经纬度模式：输入纬度和经度
- 投影模式：输入X（东向）和Y（北向）坐标
- 自动识别输入格式

**输入验证：**
- 经纬度：纬度 -90~90，经度 -180~180
- 投影坐标：正数，合理范围

**坐标转换：**
- CGCS2000投影坐标 → WGS84经纬度（使用pyproj）
- 转换后的坐标用于更新航点

### 3.4 标签页2：地图选点

**功能：**
- 点击"确定"后进入地图点击模式
- 当前航点位置在地图上标记为红色
- 用户点击地图选择新位置
- 按ESC取消选择
- 点击后显示新位置坐标预览

### 3.5 标签页3：精细移动

**功能：**
- 输入X偏移量（东西方向，正=东，负=西）
- 输入Y偏移量（南北方向，正=北，负=南）
- 单位：米
- 实时预览新位置坐标（WGS84和CGCS2000）
- 快捷按钮：±1m、±10m（东/西/南/北）

**坐标偏移计算：**
- 使用GpxEditor.offset_coordinates()方法
- 基于WGS84经纬度计算偏移后的坐标

### 3.6 实现要点

- 重写 `gpx_editor/gui/move_waypoint_dialog.py` — 使用Notebook标签页
- 修改 `gpx_editor/core/coord_converter.py` — 添加CGCS2000→WGS84转换
- 修改 `gpx_editor/gui/main_window.py` — 更新移动航点调用

## 四、功能三：地图交互编辑

### 4.1 需求描述

实现MapSource风格的地图编辑体验：
- 拖拽移动航点/航迹点
- 双击航迹点插入新点
- 右键航迹点删除
- 航迹点列表与地图联动选中
- 地图源更换为天地图（路网+卫星影像）

### 4.2 天地图集成

**瓦片服务：**
- 路网图：`http://t{s}.tianditu.gov.cn/vec_w/wmts?SERVICE=WMTS&REQUEST=GetTile&VERSION=1.0.0&LAYER=vec&STYLE=default&TILEMATRIXSET=w&FORMAT=tiles&TILEMATRIX={z}&TILEROW={y}&TILECOL={x}&tk={API_KEY}`
- 卫星影像：`http://t{s}.tianditu.gov.cn/img_w/wmts?...`（LAYER=img）
- 路网标注：`http://t{s}.tianditu.gov.cn/cia_w/wmts?...`（LAYER=cia，叠加在卫星图上）
- 子域名：t0-t7，随机选择
- 坐标系：EPSG:4326（WGS84）

**图层切换：**
- 路网图：仅显示路网底图
- 卫星影像：卫星底图 + 路网标注叠加
- 混合图层：卫星底图 + 路网标注 + 路网底图叠加

**API Key管理：**
- 首次使用时提示用户输入天地图API Key
- 保存到配置文件 `~/.gpx_editor/config.json`
- 提供设置界面修改API Key

### 4.3 航点交互

**拖拽移动：**
- tkintermapview的marker支持拖拽
- 绑定marker的移动事件，实时更新航点坐标
- 拖拽结束后记录撤销操作

**右键菜单：**
- 删除航点
- 航点属性
- 复制坐标

**列表联动：**
- 点击航点marker时，树形列表自动选中对应航点
- 树形列表选中航点时，地图自动居中并高亮marker

### 4.4 航迹点交互

**显示方式：**
- 航迹点用小号marker显示（半径4px，绿色）
- 航迹线用path绘制（绿色虚线）

**拖拽移动：**
- 拖拽航迹点到新位置
- 实时重绘航迹线
- 拖拽结束后记录撤销操作

**双击插入：**
- 双击航迹线上的任意位置
- 在最近的两个航迹点之间插入新点
- 自动重绘航迹线
- 记录撤销操作

**右键删除：**
- 右键航迹点弹出菜单
- 选择删除后移除该航迹点
- 自动重绘航迹线
- 记录撤销操作

**列表联动：**
- 点击航迹点时，树形列表自动选中对应航迹
- 树形列表选中航迹时，地图自动缩放到航迹范围

### 4.5 撤销/重做支持

所有地图编辑操作都支持撤销/重做：
- 拖拽航点：记录旧坐标和新坐标
- 拖拽航迹点：记录旧坐标和新坐标
- 插入航迹点：记录插入位置和点数据
- 删除航迹点：记录删除位置和点数据

快捷键：Ctrl+Z 撤销，Ctrl+Y 重做

### 4.6 实现要点

- 修改 `gpx_editor/gui/main_window.py` — 集成天地图、航点/航迹点交互
- 修改 `gpx_editor/gui/map_view.py` — 地图交互逻辑（如果存在）
- 新建 `gpx_editor/core/coord_converter.py` — 天地图瓦片URL生成
- 修改 `gpx_editor/gui/undo_manager.py` — 支持新的操作类型
