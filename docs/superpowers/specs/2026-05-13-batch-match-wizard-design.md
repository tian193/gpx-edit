# 批量匹配航点向导 - 设计文档

> 创建日期: 2026-05-13

## 一、概述

将现有的"批量航点匹配"对话框重构为**多步骤标签页向导**，增加灵活的删除选项和多种输入方式支持。

## 二、需求总结

### 输入方式
- 基线文件：支持单文件或文件夹（文件夹可选"合并航点"或"逐个匹配"）
- 源文件：支持单文件或文件夹
- 输出目录：统一输出目录，基线和源文件结果都输出到这里

### 匹配参数
- 距离阈值（米）：默认100
- 航点前缀（可选）：匹配后重命名时添加的前缀

### 删除选项（可多选）
- 删除未匹配的基线航点
- 删除未匹配的源航点
- 删除匹配的基线航点
- 删除匹配的源航点

### 预览功能
- 全局摘要统计（文件数、航点数、匹配数、删除数）
- 逐文件详细航点列表（源航点名、坐标、匹配基线航点、距离、状态）

## 四、核心模块修改

### BatchMatcher 类新增方法

1. `process_single_file_with_options()` — 支持删除选项的单文件处理
2. `batch_process_with_options()` — 支持删除选项的批量处理
3. `load_baseline_from_dir()` — 从文件夹加载并合并基线航点
4. `apply_baseline_deletions()` — 应用基线航点删除操作

### 数据流

```
基线文件 ──加载航点──┐
                     ├──匹配──→ 预览结果 ──应用删除选项──→ 输出文件
源文件 ────加载航点──┘
```

### 删除选项处理逻辑

匹配完成后，根据用户选择的删除选项：

1. **删除未匹配的基线航点**：从基线文件中移除没有被任何源航点匹配到的航点
2. **删除未匹配的源航点**：从源文件中移除没有匹配到基线的航点（当前已实现）
3. **删除匹配的基线航点**：从基线文件中移除被匹配到的航点
4. **删除匹配的源航点**：从源文件中移除匹配到基线的航点

基线文件修改后保存到输出目录，不修改原文件。
